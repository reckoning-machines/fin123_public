Bring Your Model

Analysts should not have to rebuild working Excel models to enter the fin123 ecosystem.

Bring Your Model is the first-class import layer for fin123. Analysts can bring the workbook they already trust: formulas, Bloomberg links, Visible Alpha links, internal data references, and model structure are preserved and mapped into the fin123 workflow.

The model keeps working, but once uploaded it gains fin123 versioning, Sheet Warnings, chat context, Audit, replay, and governed =AI() upgrades. Analysts can keep the workbook logic that already works while the firm gets less migration friction, fewer parity bugs, and a faster path from spreadsheet artifact to governed decision system.

  • Import existing Excel models instead of rebuilding them.
  • Preserve formulas, Bloomberg links, Visible Alpha links, and workbook structure.
  • Map live model inputs into approved =DATA() and governed execution paths.
  • Import comments into YAP chat for later promotion to Model Memory if approved.
  • Add versioning, Sheet Warnings, YAP context, Audit, replay, and =AI() without breaking analyst workflow.

Replay and Audit prove the workflow.

Every workflow run leaves a reviewable record: model state, company, scenario, data evidence, AI method, approved memory, results, replay context, and Audit.

Model -> Version -> Scenario -> Run -> Results -> Replay -> Audit

  • Versions preserve model states before execution.
  • Runs preserve company and scenario context.
  • Replay uses the model state, data, provider evidence, AI method, memory, chronology, and time rules that applied then.
  • Audit shows what ran, what changed, which evidence was used, and who approved it.
  • Saved model states can be scheduled for later execution without opening Excel on a server.

AI that participates in the model.

Not a side chat. Not copy/paste. =AI() lets reviewed AI workflows execute as model steps.

=AI() sits in a normal worksheet cell. The approved definition lives behind the cell, where a Method can retrieve evidence, compare history, validate assumptions, and return typed output to the model.

Audit records the method, inputs, validation, approved memory, chronology, and Run identity behind the result.

YAP and Model Memory

Desk knowledge should not disappear into chat logs, email threads, or analyst notebooks. YAP, the in-workbook team-wide chat, captures discussion. Model Memory captures approved knowledge. AI workflows can use that approved knowledge inside the model.

How Desk Knowledge Enters The Model

Desk knowledge becomes approved workflow context.

YAP is not generic chat. Discussion can become approved Model Memory, and Model Memory can inform future AI workflows. YAP is non-mutating: it does not silently change Runs, Results, assumptions, or active Model Memory.

The lifecycle that prevents one wrong number

fin123 makes model changes explicit and outputs repeatable. You always know what changed, what ran, and what produced the number you’re looking at.

Commit

Freeze model logic into an immutable version you can review and reference later. Data stays separate from the committed model.

Build

Run headlessly and produce versioned outputs as artifacts. The same model and the same local inputs produce the same outputs.

Verify

Verify that outputs match the recorded model version and input fingerprints. If anything drifts, verification fails.

Release

Promote a verified build for production use. Releases are explicit pointers to known-good artifacts — nothing updates silently.

What fin123 is (and is not)
fin123 is a spreadsheet system with discipline. Model logic, data inputs, and formatting are separate so correctness can be enforced. It is not a reporting tool or a deck generator.

System guarantees

These are the guarantees fin123 is designed to enforce — the contract teams can rely on.

Authoring that feels like modeling

Grid-based authoring for analysts, without coupling logic to presentation. Formatting stays separate from the model and never changes what the model computes.

Scenarios are first-class

Base, upside, downside, and perturbations run against the same committed model. Table materialization is reused when only scalar parameters change.

Parameters

Parameters are explicit model inputs, not hidden cell state. Builds record effective parameters, scenario overlays, and a parameter fingerprint so you can run the same model with different params. For example: same model, different "tickers".

Deterministic builds

Builds are reproducible given the same committed model and the same local inputs. Inputs are fingerprinted and builds record enough provenance to prove what was used.

Guardrails for lookups and joins

Fail fast on ambiguous joins, duplicate keys, and missing required data. Silent row multiplication is unacceptable in finance.

Offline execution with optional pre-sync

External data access is isolated to a sync step that writes local caches. Builds never reach out to live databases.

Versioned artifacts and provenance

Every output is an identifiable artifact with lineage: model version, input fingerprints, and the exact artifact identity. This supports diffing across runs and reproducible research.

No PRINT button, by design
fin123 produces artifacts for review, verification, and downstream pipelines. If you need a report, generate it from released data — don’t treat formatting as the system of record.

Fast on real data

Built on Polars query plans for speed and scale. Tables compose as optimized plans and materialize only when collected.

DB optional

Run fully standalone on the filesystem. Enable a DB-backed registry when you want team visibility and centralized execution.

Native XLSX import and export

Bring existing Excel models in as first-class fin123 models, with a reviewable import report. Export back to XLSX when you need a handoff — without turning formatting into the system of record.

Why this matters

Scale the same model across cases and names

Analysts need the same model executed across many entities and scenarios. fin123 treats this as a structured build process with traceable outputs — not a manual spreadsheet ritual.

Reproducibility you can prove

Results must be reproducible. fin123 provides objective verification so outputs can be trusted, reviewed, and reproduced without relying on a machine’s invisible state.

Separation that prevents drift

Model logic is versioned independently from vendor and market data. Data can be synced ahead of time, but evaluation remains offline. This keeps runs explainable.

Discipline applied to spreadsheets

Spreadsheet workflows often blur editing, execution, and publication. fin123 forces boundaries: commit changes, build artifacts, verify integrity, then release only what you approve.

Run it

Headless-first. Run from the terminal, from CI, or from your own tooling. The UI is optional; version discipline is not.

terminal 
fin123 commit
fin123 build --scenario base
fin123 verify-build <build_id>
